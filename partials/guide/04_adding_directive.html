Directive

  • Directives is what AngularJS refers to as "teaching HTML new tricks".
  • Directives are markers on a DOM element which attach a special behavior to it.
  • For example static HTML wont know how to create and display a modal box or a datepicker or a slideshow carousal.Using directives you can teach your HTML to behave with a specific functionality.
  • Angular has some built-in directives like ng-repeat,ng-bind,ng-show,etc

INFO:You can relate a directive to Web Components that are going to be the future of HTML. (Think of your <video></video>,<audio></audio>,<article></article>,etc as a built-in directive of HTML 5). Check out the below link to have a clear idea of web components and see how you can relate them to your directive.

Custom Directives

Some common use cases of using directives would be

  • Reusable code blocks with certain functionality can be achieved by a directive
  • Components that modify DOM or external plugins where two way data binding is to be maintained

Adding a Directive

A very basic directive would look like below

	angular.module('utils.helloWorld',[])
	.directive('proHelloWorld', function() {
    return {
    restrict: 'AEC',
    replace: 'true',
    template: '<h3>Hello World!!</h3>'
    };
});

Then in your HTML

    <pro-hello-world></pro-hello-world>

The output would look like below

Hello World!!

Ways of using a Directive

Directive can be used either as a element itself or as an attribute or as an comment

  • A new HTML element (<pro-hello-world></pro-hello-world>).
  • An attribute on an element (<div pro-hello-world/>).
  • As a class (<div class="pro-hello-world"/>).
  • As comment (<!--directive:date-pro-hello-world-->).
