Running your application

There are so many ways to serve your application for testing.Some of them are

  • using grunt serve,a preconfigured grunt task that starts a web server using node http module
  • place your application folder inside your web server public directories(e.g www/ folder if nginx or htdocs/ in apache etc. )
  • If using Brackets IDE or some other supported IDE you can directly start your testing server from the available options.

Using grunt serve

Serve is a custom task most commonly used to serve your application over http. Usually running this task starts some additional tasks to ease your developemnt such as

  • watches any changes in your assets and reloads the page automatically.(fully configurable)
  • additional tasks such as injector,ngtemplates etc can be added as a prerequisites
$ grunt serve
